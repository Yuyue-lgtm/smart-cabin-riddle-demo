# MVP 功能规格

## 版本范围

本文定义 V1.x 阶段的最小闭环。V1.0 已完成应用骨架；V1.1 之后的重点是把 demo 从“手动演示页”推进为“可分享体验版”。

V1.x 的目标：

- 用户可以通过公网链接体验一局 AI 猜谜。
- 用户只需要控制一个真实输入，其他乘客由系统模拟。
- 车速、环境、乘客状态和车辆事件通过时间轴自动发生。
- Workflow 负责 AI 主持、胜负判定、模拟乘客动作和融合决策。
- 网页负责座舱表现、游戏表现、车内互动和事件执行。

## 目标闭环

V1.x 必须稳定跑通：

```text
开局
→ AI 开场介绍
→ 用户或模拟乘客提问
→ AI 回答
→ 时间轴触发环境/乘客/车辆事件
→ 急刹等高优先级事件打断
→ 游戏暂停
→ 恢复游戏
→ 用户或模拟乘客继续提问
→ AI 回答
→ 用户或模拟乘客猜答案
→ 猜对 / 猜错 / 揭晓
```

## 页面布局

页面只考虑横屏大屏，优先适配 16:9 或车机横屏。

布局比例：

- 左侧区域1：约 44%
- 右侧区域：约 56%
- 右侧上方区域2：游戏界面
- 右侧下方区域3：多模态感知与控制区

当前不考虑手机端适配。

## 区域1：虚拟座舱与车内互动区

### 用途

区域1负责表现“座舱里发生了什么”，并承载用户与模拟乘客的车内互动。

### V1.x 必备内容

座舱表现：

- 车外环境背景
- 2D 插画式座舱占位
- 四个座位：
  - 主驾
  - 副驾
  - 后排左
  - 后排右
- 每个乘客支持：
  - 当前情绪
  - 当前发言气泡
  - 被 AI cue 到时的高亮
- 座舱状态提示：
  - 游戏进行中
  - 安全暂停中
  - 主驾专注模式
  - 轻声互动模式
  - 胜利氛围
- 答对反馈：
  - 答对乘客座位氛围灯亮起
  - 播放答对音效

车内互动区：

- 位于左侧区域底部
- 显示当前发言座位
- 提供用户输入框
- 提供发送提问/答案按钮
- 展示最近的模拟乘客发言
- 可展示简短车内对话记录

### 乘客情绪

V1.x 支持：

- 普通
- 大笑
- 沉默
- 睡着

### 车外环境

V1.x 支持：

- 高速路晴天白天
- 高速路晴天深夜
- 高速路雨天白天
- 城区晴天白天
- 城区雨天白天
- 风景区晴天白天
- 风景区雪景白天

当前环境图先用占位表现，后续可替换为插画、真实图或生成图。

## 区域2：游戏界面

### 用途

区域2负责表现猜谜游戏流程和 AI 主持人。

### V1.x 必备内容

- 当前游戏进度，例如 `1/10`
- 当前提问进度，例如 `2/15`
- 当前游戏状态：
  - 待开局
  - 开场
  - 进行中
  - 已暂停
  - 猜对
  - 揭晓
- AI 主持人占位形象
- AI 主持人话术气泡
- 当前谜底展示区
- 谜底默认隐藏，猜对/失败/揭晓时显示
- 猜对时播放简短正反馈音效

### 游戏规则

- AI 主持人出题
- 用户和模拟乘客轮流问“是/否”问题
- 15 个问题内猜出谜底
- 10 道题可进行一次结算
- 主持人可根据车程情况提前或延后结算

V1.x 先实现单题闭环和 10 题进度展示，完整 10 题结算后续迭代。

## 区域3：多模态感知与控制区

### 用途

区域3用于模拟车速、目的地、环境、乘客关系、乘客状态和车辆事件。它既可以手动控制，也可以通过时间轴自动变化。

### V1.x 必备控制项

车辆相关：

- 车速
- 目的地
- 车外环境

乘客相关：

- 乘客关系
- 乘客状态

时间轴相关：

- 当前时间轴名称
- 重置模拟 / 切换下一条时间轴
- 时间轴开始/暂停
- 当前时间轴事件提示

事件相关：

- 急刹打断
- 主驾疲惫
- 有人睡着
- 快到目的地
- 恢复游戏

Workflow 相关：

- Workflow 接口配置
- 请求状态
- 必要时显示最近一次错误

### 控制区变化

原 V1.0 底部输入栏后续应迁移到左侧区域底部。右下区域不再承载主要玩家输入，而是专注于多模态感知和事件控制。

## 时间轴模拟

### 目标

时间轴用于模拟车程中多模态感知信息自动发生，让体验不依赖演示者频繁手动点击。

每次点击“重置模拟”时，系统选择或切换一条预设时间轴，并从头开始执行。

### 时间轴事件类型

时间轴可以触发：

- 车速变化
- 目的地变化
- 车外环境变化
- 乘客关系变化
- 乘客情绪变化
- 主驾疲惫
- 有人睡着
- 急刹打断
- 恢复游戏
- 快到目的地
- 请求模拟乘客发言

### 时间轴示例

```text
时间轴 A：高速亲子出行
0s：高速路晴天白天，车速 80，目的地迪士尼
10s：AI 开场
18s：后排小朋友尝试提问
30s：主驾疲惫
45s：急刹打断
70s：恢复游戏
95s：快到目的地
```

```text
时间轴 B：雨天朋友聚会
0s：城区晴天白天，车速 50，目的地火锅店
20s：切换为城区雨天白天
32s：副驾大笑
48s：左后模拟乘客提问
65s：玩家接近答案
```

### 时间轴规则

- 高优先级事件可以打断当前游戏状态。
- 游戏暂停时，模拟乘客不应继续提问。
- 手动事件按钮可以覆盖当前时间轴状态。
- 时间轴事件必须进入统一状态中心，再驱动 UI。

## 模拟乘客系统

### 目标

模拟乘客用于提升分享体验的真实感。用户不需要自己扮演全车人，系统会让其他乘客半自主参与。

### 实现原则

第一阶段不做多个独立外部 Agent。采用：

```text
乘客人设
→ 发言意图池
→ 游戏状态判断
→ 随机选择
→ Workflow 改写
→ passenger_action 输出
```

### 乘客人设

每个座位可以配置：

- 座位
- 名称
- 人设
- 发言风格
- 参与频率
- 是否允许主动猜答案

示例：

```json
{
  "seat": "rearRight",
  "name": "后排小朋友",
  "persona": "好奇、兴奋、喜欢动画和食物",
  "style": "短句、直接、偶尔抢答",
  "can_guess": true
}
```

### 发言意图

V1.x 建议先支持：

- `ask_category`：问类别
- `ask_attribute`：问属性
- `ask_scene`：问使用场景
- `guess_answer`：猜答案

后续可扩展：

- `react_to_host`：回应主持人
- `joke`：玩梗
- `cue_user`：带动真实用户参与

### 真实感约束

模拟乘客必须遵守：

- 不在游戏暂停时发言
- 不连续让同一个模拟乘客发言太多次
- 不重复问已经问过的问题
- 不在太早阶段直接猜中
- 睡着的乘客不主动发言
- 主驾在高速或复杂路况下降低参与
- 小朋友问题更短更简单
- 中老年乘客话术更稳重
- 年轻朋友可以更活泼

### passenger_action

Workflow 第一阶段只需要支持单个模拟乘客动作：

```json
{
  "passenger_action": {
    "seat": "rearRight",
    "text": "它是不是车里能保护我们的东西？",
    "mood": "普通"
  }
}
```

网页收到后：

- 对应乘客头顶冒气泡
- 左侧底部互动区追加发言
- 更新对应乘客情绪
- 再展示 AI 主持人的回复

## 事件优先级

事件采用打断机制。高优先级事件可以中断当前游戏流程。

P0：

- 急刹
- 激烈变道
- 车辆风险状态

P1：

- 主驾疲惫
- 高速高车速
- 有人接打电话
- 恢复游戏

P2：

- 有人睡着
- 快到目的地
- 天气变化
- 时间轴普通事件

P3：

- 普通玩家提问
- 模拟乘客提问
- 乘客情绪变化
- 乘客关系变化

### 急刹行为

急刹触发后，网页必须立即：

- 游戏状态切换为暂停
- 座舱显示安全暂停
- AI 话术切到安全提醒
- 暂时阻止用户继续提问
- 暂停模拟乘客主动发言

Workflow 返回后，可补充更自然的话术和 UI 指令。

## Workflow 与 Mock 定位

### Workflow 主路径

正式体验以 Coze/Dify Workflow 为主路径。

Workflow 负责：

- AI 主持人话术
- 猜谜胜负判定
- 模拟乘客动作生成
- 多模态状态融合决策
- 高优先级事件策略
- 结构化 JSON 输出

### Mock 弱化

Mock 只作为开发期或异常保护，不作为对外体验重点。

对外分享版中，如果 Workflow 不可用，优先提示：

```text
AI 服务暂时繁忙，请稍后再试。
```

开发环境可以保留 Mock 开关，方便调试页面状态。

## 插件化边界

猜谜是当前游戏插件。

插件负责：

- 游戏状态
- 题库/谜底
- 玩家与模拟乘客输入解释
- 胜负判定结果应用
- 游戏区域展示内容

座舱系统负责：

- 车辆状态
- 环境状态
- 乘客状态
- 时间轴事件
- 高优先级事件
- Workflow 输入输出适配
- 模拟乘客通用约束

未来插件：

- 成语接龙
- 观察挑战
- 找找看
- 猜歌
- 故事接龙

## 暂不做事项

V1.x 暂不做：

- 多个独立外部 Agent 并行
- 真正多人在线
- 真实语音播放
- 真实多模态传感器接入
- 复杂动画系统
- 手机端适配
- 完整 10 题结算页
- 用户账号与长期记忆
- 完整排行榜
- 后台题库管理

这些能力可以进入 V2.x 或比赛后优化阶段。

- 答对乘客座位氛围灯亮起
- 答对音效
- 胜利状态动画
- 安全暂停状态动画
