# 公网端到端验收记录

## 验收目标

确认公网分享版本可以稳定完成一条比赛演示主链路：

```text
开始模拟 -> AI 主持开场 -> 玩家提问 -> AI 回答 -> 急刹打断 -> 自动恢复 -> 再提问 -> 猜对 -> 谜底揭晓
```

这一步不追求新增功能，重点是找出会影响比赛演示和部门分享体验的稳定性问题。

## V1.2 本地验收（2026-06-21）

本轮使用 Playwright + 系统 Chrome 对本地页面执行自动验收，并通过本地代理验证真实 Coze Workflow。

前端通过项：

- 中控屏内容区 16:9，座舱位于右侧
- 固定副驾输入，无座位下拉框
- 三条黄金体验线可切换
- 开始模拟、暂停/继续、暂停时重置均可用
- 普通副驾提问与模拟乘客气泡正常
- 猜对后谜底揭晓、答对座位氛围灯与三音提示正常
- 急刹立即暂停，随后自动恢复
- `1280 x 500` 矮屏下完整舞台可见，无上下裁切
- 浏览器控制台无页面错误

真实 Workflow 结果：

| 请求 | HTTP | 耗时 | 结果 |
| --- | --- | --- | --- |
| `start_game` | 200 | 6.27s | 开场和后排模拟乘客返回正常 |
| 副驾普通提问 | 200 | 8.51s | AI 回答和后排模拟乘客返回正常 |
| 副驾猜对 | 200 | 7.89s | `is_correct=true`、`game_status=victory`、谜底返回正常 |

本轮修复：

- Workflow 在猜对时仍返回后排 `passenger_action` 和错误 `target_seat`。前端现以胜利状态优先：忽略模拟乘客动作、清除模拟气泡，并将答对座位锁定为本次真实发言者。
- 补充答对三音提示，不引入外部音频资源。

待优化但不阻断 V1.2：

- Coze 猜对话术出现“副驾的朋友”表达，建议在 Workflow 主持人提示词中改为直接称呼“副驾”。
- Coze 当前响应约 6-9 秒，后续应继续优化节点长度和模型调用次数。
- Workflow 猜对时仍生成 `passenger_action`；前端已兜底，Workflow 侧仍应在 `is_correct=true` 时禁止生成。

## 当前界面基准

V1.2 之后的当前界面基准为：

- 左侧：中控屏 + 手动控制区
- 右侧：座舱与环境模拟区
- 中控屏内容区保持 16:9，标题栏不计入比例
- 真实用户固定为副驾，不提供座位下拉选择
- `场景切换` 用于切换黄金体验线
- 急刹后前端自动恢复，不再提供手动 `恢复游戏` 按钮
- 猜对后答对乘客座位氛围灯亮起

## 准入标准

可以进入下一阶段开发，需要满足：

- P0 问题数量为 0
- P1 问题数量不超过 2 个，且有明确规避方式
- 主链路可以连续跑通 3 次
- 急刹打断和自动恢复不能出现状态错乱
- Coze 返回异常时页面不能白屏或卡死

## 问题优先级

| 优先级 | 定义 | 例子 | 处理要求 |
| --- | --- | --- | --- |
| P0 | 阻断演示或分享体验 | 页面打不开、开始模拟失败、急刹后无法恢复、Token 泄露风险 | 立即修复 |
| P1 | 影响核心体验但可绕过 | Coze 偶发慢、话术不符合场景、状态局部不同步 | 本阶段修复或给出规避方案 |
| P2 | 体验瑕疵 | 动画不够顺、文案不够自然、局部样式不够精致 | 进入后续优化 |

## 主链路验收

### 1. 页面加载

步骤：

1. 打开公网链接
2. 设置浏览器为 16:9 横屏尺寸
3. 等待页面完全加载

预期：

- 三分区布局正常
- 左上中控屏可见
- 左下手动控制区可见
- 右侧座舱与环境模拟区可见
- 底部副驾输入区不遮挡控制项
- 页面没有明显横向或纵向滚动条

结果：通过

### 2. 开始模拟

步骤：

1. 点击 `开始模拟`
2. 等待 Coze 返回

预期：

- 游戏从待开始进入进行中
- AI 主持人气泡展示开场话术
- 话术能结合当前环境、车速、乘客关系或谜题主题
- 如果返回了 `passenger_action`，对应乘客会出现气泡

结果：通过

### 3. 玩家提问

步骤：

1. 保持当前选中乘客为 `副驾`
2. 输入：

```text
它是车上的东西吗？
```

3. 点击 `发送提问`

预期：

- 副驾头顶出现用户提问气泡
- 提问数增加
- AI 主持人回答“是/否/不确定但可解释”的主持式话术
- 游戏仍为进行中
- 页面没有重复提交、卡死或状态回退

结果：通过

### 4. 急刹打断

步骤：

1. 点击 `急刹打断`

预期：

- 前端立即进入暂停态，不等待 Coze 返回
- AI 主持人进入安全提醒语气
- 乘客模拟发言停止
- 继续发送普通提问应被阻止或提示暂停中
- 前端按合理时间自动恢复，Coze 返回只补充话术和状态

结果：通过

### 5. 自动恢复游戏

步骤：

1. 急刹后等待前端自动恢复

预期：

- 游戏从暂停态恢复为进行中
- AI 主持人给出自然的恢复话术
- 原谜底、提问数、历史上下文不丢失
- 玩家可以继续提问

结果：通过

### 6. 再次提问

步骤：

1. 输入：

```text
它和安全有关吗？
```

2. 点击 `发送提问`

预期：

- 提问数继续增加，而不是重新开始
- AI 回答能承接急刹前后的上下文
- 乘客气泡和互动记录正常

结果：通过

### 7. 猜对

步骤：

1. 输入：

```text
答案是安全带
```

2. 点击 `发送提问`

预期：

- Workflow 返回 `is_correct = true`
- 游戏状态进入猜对或胜利态
- 谜底揭晓为 `安全带`
- AI 主持人给出胜利/MVP/情绪价值话术
- 答对乘客座位氛围灯亮起
- 播放答对音效

结果：基础流程通过；座位氛围灯和答对音效仍按路线图进入 V1.4 UI 与反馈优化
