# 公网端到端验收记录

## 验收目标

确认公网分享版本可以稳定完成一条比赛演示主链路：

```text
AI 开场 -> 玩家提问 -> AI 回答 -> 急刹打断 -> 恢复游戏 -> 再提问 -> 猜对 -> 谜底揭晓
```

这一步不追求新增功能，重点是找出会影响比赛演示和部门分享体验的稳定性问题。

V1.x 暂不验收手机竖屏。

## 准入标准

可以进入下一阶段开发，需要满足：

- P0 问题数量为 0
- P1 问题数量不超过 2 个，且有明确规避方式
- 主链路可以连续跑通 3 次
- 急刹打断和恢复游戏不能出现状态错乱
- Coze 返回异常时页面不能白屏或卡死

## 问题优先级

| 优先级 | 定义 | 例子 | 处理要求 |
| --- | --- | --- | --- |
| P0 | 阻断演示或分享体验 | 页面打不开、AI 开场失败、急刹后无法恢复、Token 泄露风险 | 立即修复 |
| P1 | 影响核心体验但可绕过 | Coze 偶发慢、话术不符合场景、状态局部不同步 | 本阶段修复或给出规避方案 |
| P2 | 体验瑕疵 | 动画不够顺、文案不够自然、局部样式不够精致 | 进入后续优化 |

## 主链路验收

### 1. 页面加载

步骤：

1. 打开公网链接
2. 设置浏览器为 16:9 横屏尺寸
3. 等待页面完全加载

预期：

- 三分区布局正常
- 左侧座舱区可见
- 右上游戏区可见
- 右下控制区可见
- 左侧底部车内互动输入区不遮挡座舱主体
- 页面没有明显横向或纵向滚动条

结果：待测

### 2. AI 开场

步骤：

1. 点击 `AI 开场`
2. 等待 Coze 返回

预期：

- 游戏状态从 `待开局` 进入 `进行中`
- AI 主持人气泡展示开场话术
- 话术能结合当前环境、车速、乘客关系或谜题主题
- 如果返回了 `passenger_action`，对应乘客会出现气泡

结果：待测

### 3. 玩家提问

步骤：

1. 保持当前选中乘客为 `副驾`
2. 输入：

```text
它是车上的东西吗？
```

3. 点击 `发送提问`

预期：

- 副驾头顶出现用户提问气泡
- 提问数增加
- AI 主持人回答“是/否/不确定但可解释”的主持式话术
- 游戏仍为进行中
- 页面没有重复提交、卡死或状态回退

结果：待测

### 4. 急刹打断

步骤：

1. 点击 `急刹打断`

预期：

- 前端立即进入暂停态，不等待 Coze 返回
- AI 主持人进入安全提醒语气
- 乘客模拟发言停止
- 继续发送普通提问应被阻止或提示暂停中
- Coze 返回后页面仍保持暂停态

结果：待测

### 5. 恢复游戏

步骤：

1. 点击 `恢复游戏`

预期：

- 游戏从暂停态恢复为进行中
- AI 主持人给出自然的恢复话术
- 原谜底、提问数、历史上下文不丢失
- 玩家可以继续提问

结果：待测

### 6. 再次提问

步骤：

1. 输入：

```text
它和安全有关吗？
```

2. 点击 `发送提问`

预期：

- 提问数继续增加，而不是重新开始
- AI 回答能承接急刹前后的上下文
- 乘客气泡和互动记录正常

结果：待测

### 7. 猜对

步骤：

1. 输入：

```text
答案是安全带
```

2. 点击 `发送提问`

预期：

- Workflow 返回 `is_correct = true`
- 游戏状态进入猜对或胜利态
- 谜底揭晓为 `安全带`
- AI 主持人给出胜利/MVP/情绪价值话术
- 答对乘客座位氛围灯亮起
- 播放答对音效

结果：待测

| 有人睡着 | 将某乘客状态设为睡着 | AI 不 cue 睡着乘客，模拟乘客不选择该座位 | 待测 |
| 快到目的地 | 点击快到目的地事件 | AI 进入收尾或绝杀局话术 | 待测 |

| 检查项 | 预期 | 结果 |
| --- | --- | --- |
| JSON 可解析 | 每次返回都能被前端解析 | 待测 |
| 字段完整 | 至少包含 `ai_reply_text` 和 `game_status` | 待测 |
| 正确判定 | 猜中时返回 `is_correct: true` | 待测 |
| UI 指令可执行 | `ui_change` 不导致前端异常 | 待测 |
| 模拟乘客 | `passenger_action` 最多一个，且不选睡着乘客 | 待测 |
| 话术自然 | 不像模板拼接，能体现眼力见和情绪价值 | 待测 |
