# 黄金体验线

## 目标

黄金体验线用于把 demo 从“功能按钮集合”变成“可稳定复现的智能座舱故事”。每条线都应让体验者看见：

```text
座舱感知 -> 融合决策 -> AI 主持控场 -> 网页执行反馈
```

V1.2 先完成 3 条黄金体验线设计，并优先实现第 1 条。每条线都要能同时服务比赛现场演示和部门同事自主体验。

## 共通规则

- 默认真实用户座位为 `副驾`。
- 其他乘客由系统模拟，不接外部多 Agent。
- 每条线控制在 90-150 秒内，不追求完整 10 题结算。
- 高优先级安全事件必须立即响应，不等待 Workflow。
- 每条线至少包含一次多模态事件、一次模拟乘客参与、一次 AI 控场和一次 UI 高光。
- 时间轴事件自动发生，但保留右下控制区的手动事件触发作为备用。
- 目的地和乘客关系由体验线初始化，不在右下控制区手动切换。
- Workflow 输出建议包含 `decision_trace`，供左侧座舱区的融合决策可视化面板展示。
- 用户输入栏固定在左侧座舱底部，任何时候不置灰；Workflow 忙时输入排队。

## 当前实现状态

V1.2 第一阶段已优先实现体验线 A 的前端自动时间轴雏形：

- 默认初始化为高速亲子出行。
- 自动推进 AI 开场、后排小朋友提问、副驾提问、主驾疲惫、急刹、恢复游戏、再次提问和猜对收尾。
- 融合决策摘要显示在左侧座舱区。
- 右下控制区只保留 Workflow、时间轴、车速、车外环境和事件控制。
- 目的地和乘客关系随体验线初始化，不再手动切换。
- 用户输入可随时提交；当 Workflow 正忙时，输入进入前端队列。

## 体验线 A：高速亲子出行

### 一句话故事

一家人开车去迪士尼，后排小朋友积极参与猜谜。途中主驾疲惫和急刹打断出现，AI 主持人从游戏主持切换为安全控场，再恢复游戏并完成高情绪价值收尾。

### 展示重点

- 高速安全优先级
- 主驾疲惫时降低主驾互动
- 急刹时立即暂停游戏
- 后排小朋友更跳脱、更好奇
- 恢复后 AI 保留上下文继续主持
- 猜对后给予小朋友或副驾高光

### 默认配置

| 项目 | 值 |
| --- | --- |
| 时间轴 ID | `family_highway_disney` |
| 乘客关系 | `父母+小孩` |
| 真实用户座位 | `副驾` |
| 车外环境 | `高速路晴天白天` |
| 初始车速 | `80 km/h` |
| 目的地 | `迪士尼` |
| 推荐谜底 | `安全带` 或 `米老鼠` |
| 推荐主谜底 | `安全带` |

### 乘客人设

| 座位 | 人设 | 发言风格 |
| --- | --- | --- |
| 主驾 | 家长，专注驾驶 | 尽量少发言，高速时不主动参与 |
| 副驾 | 真实用户，主要玩家 | 负责关键提问和猜答案 |
| 后排左 | 家庭成员，辅助型 | 问题稳一点，偶尔补充 |
| 后排右 | 小朋友 | 更跳脱、更好奇，短句，容易联想到动画和游乐园 |

### 时间轴脚本

| 时间 | 事件 | 输入/状态 | AI 主持策略 | UI 高光 |
| --- | --- | --- | --- | --- |
| 0s | 初始化 | 高速晴天，目的地迪士尼，车速 80 | 准备开局 | 左侧显示高速环境和家庭乘客 |
| 5s | 首屏提示 | 显示轻提示 | 引导副驾点击 AI 开场 | 提示文案轻量出现 |
| 10s | AI 开场 | `event.type = start_game` | 开场并说明 15 问规则 | 右上游戏状态进入开场 |
| 20s | 模拟乘客参与 | 后排右小朋友提问 | 回应小朋友，夸具体 | 后排右气泡弹出 |
| 35s | 用户提问 | 副驾问“它和安全有关吗？” | 判断方向接近，制造推进感 | 副驾高亮 |
| 50s | 主驾疲惫 | `event.type = driver_tired` | 降低主驾参与，副驾和后排接管 | 融合决策面板显示 S03 |
| 65s | 急刹打断 | `event.type = hard_brake` | 立即暂停，安全提醒 | 座舱进入安全暂停 |
| 85s | 恢复游戏 | `event.type = resume_game` | 温柔恢复，承接刚才上下文 | 游戏恢复进行中 |
| 100s | 小朋友跳脱提问 | 后排右问“它是不是像超人一样保护我们？” | 接住童趣并回答方向 | 后排右气泡，AI 轻松回应 |
| 120s | 猜对 | 副驾或后排右输入“答案是安全带” | 揭晓答案，给 MVP 总结 | 答对座位氛围灯、音效、AI 总结 |

### 融合决策面板示例

```json
{
  "perception": "检测到主驾疲惫，当前车速 80km/h",
  "decision": "降低主驾参与，副驾和后排接管提问",
  "execution": "切换主驾专注模式，AI cue 副驾和后排",
  "strategy_id": "S03",
  "priority": "P1"
}
```

### 关键 AI 话术方向

- 开场：目的地和家庭氛围要进话术，但不要直接泄题。
- 主驾疲惫：关心但不指责，明确不让主驾答题。
- 急刹：短句、坚定、安全优先，不玩梗。
- 恢复：说明“刚才上下文还在”，增强智能感。
- 小朋友答对：给称号，夸具体。

### 备用手动触发

| 目标 | 手动操作 |
| --- | --- |
| 触发主驾疲惫 | 点击 `主驾疲惫` |
| 触发急刹 | 点击 `急刹打断` |
| 恢复游戏 | 点击 `恢复游戏` |
| 触发小朋友参与 | 选择 `右后` 后输入一条童趣问题 |
| 快速收尾 | 选择答对座位，输入 `答案是安全带` |

## 体验线 B：雨天朋友聚会

### 一句话故事

年轻朋友开车去火锅店，天气从晴天变成雨天。AI 主持人把雨天融入谜题和话术，朋友之间轻松玩梗，AI 顺势接梗但不失控，最后通过提示完成猜谜。

### 展示重点

- 天气变化融入游戏
- 年轻朋友关系下更活泼的主持风格
- 模拟乘客接梗但不抢用户主角
- 玩家接近答案时增加戏剧感
- 多轮无进展时 AI 主动给提示

### 默认配置

| 项目 | 值 |
| --- | --- |
| 时间轴 ID | `rainy_city_hotpot_friends` |
| 乘客关系 | `年轻朋友` |
| 真实用户座位 | `副驾` |
| 初始环境 | `城区晴天白天` |
| 变化后环境 | `城区雨天白天` |
| 初始车速 | `50 km/h` |
| 目的地 | `火锅店` |
| 推荐谜底 | `雨伞` 或 `火锅` |
| 推荐主谜底 | `雨伞` |

### 乘客人设

| 座位 | 人设 | 发言风格 |
| --- | --- | --- |
| 主驾 | 朋友，负责开车 | 不高频参与，偶尔被 AI 保护 |
| 副驾 | 真实用户，主要玩家 | 负责推动问题 |
| 后排左 | 爱吐槽的朋友 | 会玩梗，但不攻击人 |
| 后排右 | 脑洞型朋友 | 猜测跳跃，容易带来笑点 |

### 时间轴脚本

| 时间 | 事件 | 输入/状态 | AI 主持策略 | UI 高光 |
| --- | --- | --- | --- | --- |
| 0s | 初始化 | 城区晴天，目的地火锅店 | 准备轻松朋友局 | 城区背景 |
| 8s | AI 开场 | `event.type = start_game` | 活泼开场，带一点朋友局语气 | 主持人气泡 |
| 22s | 用户提问 | 副驾问“它和天气有关吗？” | 回答并鼓励方向 | 副驾气泡 |
| 35s | 天气变化 | 环境切到城区雨天白天 | 雨天入题，强调应景 | 背景切雨天，决策面板显示 S07 |
| 48s | 模拟乘客玩梗 | 后排左吐槽一个离谱答案 | AI 接梗但拉回游戏 | 后排左气泡 |
| 65s | 多轮无进展 | 问题数接近 8 或策略触发 | AI 给轻提示，降低挫败感 | 游戏区提示动画 |
| 82s | 玩家接近答案 | 副驾问“它是不是下雨会用？” | 增加戏剧感，不直接泄题 | 主持人兴奋态 |
| 100s | 猜对 | 输入“答案是雨伞” | 揭晓答案，轻松总结 | 胜利氛围 |

### 融合决策面板示例

```json
{
  "perception": "车外环境切换为城区雨天白天",
  "decision": "将天气变化融入当前谜题提示",
  "execution": "切换雨天座舱背景，AI 用雨天话术推进游戏",
  "strategy_id": "S07",
  "priority": "P2"
}
```

### 关键 AI 话术方向

- 雨天：自然应景，不要像天气播报。
- 玩梗：接一下就拉回来，不让游戏变成纯聊天。
- 提示：用自嘲降低压力，比如“不是你们不行，是我这题有点阴险”。
- 胜利：朋友局可以更夸张一点，但不要太长。

### 备用手动触发

| 目标 | 手动操作 |
| --- | --- |
| 切雨天 | 点击 `城区雨天白天` |
| 触发玩梗 | 选择后排座位输入离谱猜测 |
| 触发提示 | 连续输入几条不接近的问题 |
| 快速收尾 | 输入 `答案是雨伞` |

## 体验线 C：风景区家庭出游

### 一句话故事

一家人去风景区，窗外切到雪景。AI 将雪景融入游戏，长辈稳健提问推动进展，途中有人睡着，AI 自动轻声继续并避免打扰，快到目的地时进入绝杀局收尾。

### 展示重点

- 风景/雪景与谜题融合
- 中老年 + 儿女关系下更稳重的主持风格
- 睡着乘客不再被 cue，不再主动发言
- 快到目的地时主动收束游戏
- 全车协作或长辈答对时有尊重感

### 默认配置

| 项目 | 值 |
| --- | --- |
| 时间轴 ID | `scenic_snow_family` |
| 乘客关系 | `中老年+儿女` |
| 真实用户座位 | `副驾` |
| 车外环境 | `风景区雪景白天` |
| 初始车速 | `50 km/h` |
| 目的地 | `草原` 或 `风景区` |
| 推荐谜底 | `雪人` 或 `草原` |
| 推荐主谜底 | `雪人` |

### 乘客人设

| 座位 | 人设 | 发言风格 |
| --- | --- | --- |
| 主驾 | 家庭成员，专心驾驶 | 少参与，避免被频繁 cue |
| 副驾 | 真实用户，辅助推进 | 问清晰问题 |
| 后排左 | 长辈 | 稳重、经验派、问题直接 |
| 后排右 | 年轻家人 | 配合气氛，可能中途睡着 |

### 时间轴脚本

| 时间 | 事件 | 输入/状态 | AI 主持策略 | UI 高光 |
| --- | --- | --- | --- | --- |
| 0s | 初始化 | 风景区雪景白天，车速 50 | 准备风景区家庭局 | 雪景背景 |
| 8s | AI 开场 | `event.type = start_game` | 用雪景入题，语气稳一点 | 主持人气泡 |
| 22s | 长辈提问 | 后排左问“它是不是冬天常见？” | 尊重回应，推进范围 | 后排左气泡 |
| 38s | 用户提问 | 副驾问“它怕热吗？” | 判断接近答案，制造戏剧感 | 副驾高亮 |
| 55s | 有人睡着 | 后排右状态变睡着 | 轻声继续，不 cue 睡着乘客 | 后排右睡着贴图，决策面板显示 S04 |
| 75s | 模拟乘客轮转 | 后排左继续稳健提问 | 不选择睡着乘客，长辈继续参与 | 后排左高亮 |
| 95s | 快到目的地 | `event.type = near_destination` | 收束节奏，进入绝杀局 | final_round 状态 |
| 115s | 猜对 | 后排左或副驾输入“答案是雪人” | 尊重感总结或全车协作总结 | 胜利氛围 |

### 关键 AI 话术方向

- 雪景：有画面感，但不要过度抒情。
- 长辈：夸稳、夸判断，不用过度玩梗。
- 睡着：体贴，明确“不打扰 TA”。
- 快到目的地：主动收尾，制造绝杀局。
- 胜利：如果长辈答对，强调经验和判断；如果全车协作，强调团队感。

## V1.2 实现优先级

建议实现顺序：

1. 先实现体验线 A：高速亲子出行。状态：前端自动时间轴雏形已完成。
2. 抽象时间轴事件结构，让 B/C 只需要配置数据。
3. 补 B/C 两条时间轴。
4. 优化 Workflow 对体验线 A 的话术稳定性和 `decision_trace` 质量。
5. 再补充答对音效、氛围灯细节和胜利总结动效。

第一阶段验收只要求体验线 A 稳定自动跑通；B/C 可以先作为文档和手动演示脚本存在。
