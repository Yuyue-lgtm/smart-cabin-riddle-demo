# 测试与体验脚本

## 使用目的

每次修改代码、接入 Workflow、调整 UI、增加时间轴或模拟乘客能力后，都按本文检查一遍，避免后续功能变多后出现状态错乱、流程断裂或体验失真。

## 测试环境

推荐横屏窗口：

- 1280 x 720
- 1920 x 1080

V1.x 不测试手机端。V1.2 起页面按 `1920 x 1080` 固定舞台设计，并按浏览器宽度等比缩放；验收重点是横屏下整体不变形、不局部错位。

公网验收：

```text
https://smart-cabin-riddle-demo.vercel.app
```

公网验收结果记录在 [PUBLIC_ACCEPTANCE_REPORT.md](./PUBLIC_ACCEPTANCE_REPORT.md)。

本地运行：

```bash
node dev-server.cjs
```

访问：

```text
http://localhost:4173
```

本地健康检查：

```text
http://localhost:4173/api/health
```

## 快速冒烟测试

用于每次小改后快速确认没有大问题。

### 1. 页面加载

步骤：

1. 打开页面
2. 观察三分区是否正常显示

预期：

- 左上显示中控屏，内容区比例为 16:9
- 左下显示手动控制区
- 右侧显示座舱与环境模拟区
- 右侧显示融合决策面板
- 手动控制区显示轻量版本状态，例如 `本地 · V1.3-public-stability · 代理就绪`
- 手动控制区不再出现目的地、乘客关系、Workflow 代理配置
- 手动控制区底部固定显示 `副驾` 输入，不是下拉框
- 无明显遮挡
- 页面不出现滚动条
- 页面宽度撑满浏览器，整体按比例缩放

### 2. 开始模拟

步骤：

1. 点击 `开始模拟`

预期：

- 游戏从 `游戏待开始` 进入进行中
- AI 主持人气泡出现开场话术
- 主持人话术面向车内乘客，不出现“点击按钮”“时间轴”等出戏表达
- 谜底仍隐藏

### 3. 用户提问

步骤：

1. 手动控制区底部固定显示 `副驾`
2. 在输入框输入：`它是车上的东西吗？`
3. 点击 `发送提问`

预期：

- 副驾头顶出现气泡
- 提问数从 `0/15` 变为 `1/15`
- AI 主持人给出回答
- 游戏状态保持 `进行中`

### 4. 模拟乘客动作

步骤：

1. 触发一次模拟乘客请求，或等待时间轴触发模拟乘客发言

预期：

- Workflow 返回 `passenger_action`
- 对应乘客头顶出现气泡
- AI 主持人对该发言作出回应
- 不应出现睡着乘客主动发言
- 不应在暂停状态下出现模拟乘客主动提问
- 不应由 Workflow 模拟 `副驾` 发言，副驾只代表真实用户

### 5. 时间轴事件

步骤：

1. 点击 `场景切换` 切换黄金体验线
2. 点击 `开始模拟`
3. 等待至少一个事件发生

预期：

- 页面显示当前黄金体验线名称
- 车速、环境、乘客状态或车辆事件自动变化
- 事件变化会同步到座舱区和 Workflow 输入

### 6. 急刹打断

步骤：

1. 点击 `急刹打断`，或等待时间轴触发急刹

预期：

- 游戏状态变为 `已暂停`
- AI 主持人说安全提醒
- 除 `继续模拟` 和 `重置模拟` 外，其他控制不可用
- 模拟乘客不再主动发言
- 急刹后前端会自动恢复游戏状态

### 7. 暂停与重置

步骤：

1. 点击 `暂停模拟`
2. 观察按钮变为 `继续模拟`
3. 点击 `重置模拟`

预期：

- 暂停后除 `继续模拟` 和 `重置模拟` 外其他手动控制不可用
- `重置模拟` 在暂停状态下仍可点击
- 重置后当前黄金体验线回到待开始状态

### 8. 忙时输入排队

步骤：

1. 在 Workflow 正在返回时，立刻输入另一条问题
2. 点击 `发送提问`

预期：

- 输入框和发送按钮不置灰
- 当前座位气泡和互动记录能正常更新
- 新输入不会丢失
- 上一轮 AI 回复完成后，队列中的问题继续提交
- 不出现 AI 连续自发多条无关回复

### 9. 猜对答案

步骤：

1. 输入：`答案是安全带`
2. 点击 `发送提问`

预期：

- 游戏状态变为 `猜对`
- 谜底显示为 `安全带`
- AI 主持人给出 MVP 或胜利话术
- 对应乘客座位氛围灯亮起
- 播放答对音效

## 分享体验脚本

用于模拟部门同事打开链接后的体验。

### 体验目标

用户无需演示者辅助，也能完成一局 AI 猜谜。

### 推荐流程

1. 打开页面
2. 观察默认座舱和当前时间轴
3. 点击 `开始模拟`
4. 用户提问一次
5. 等待或触发模拟乘客参与
6. 等待时间轴触发一次环境或车辆状态变化
7. 继续提问或猜答案
8. 完成一局

### 预期体验

- 用户知道自己可以输入问题或答案
- 其他乘客会自然参与
- AI 主持人不会只回复用户，也会回应模拟乘客
- 时间轴事件让车程有变化
- 事件打断不会导致游戏状态崩溃

## 比赛讲解脚本

适合比赛现场 3 分钟展示。

### 开场讲法

```text
这是一个可分享体验的智能座舱 AI 猜谜 Demo。用户可以作为车内乘客参与游戏，其他乘客由系统模拟。车速、环境、乘客状态和车辆事件会通过时间轴自动发生，AI 主持人基于这些上下文控场。
```

### 演示步骤

1. 展示三分区

```text
左边是中控屏和手动控制区，右边是座舱与环境模拟区。中控屏负责游戏主持，右侧座舱负责表现乘客、环境和融合决策。
```

2. 点击 `开始模拟`

```text
AI 会根据当前座舱状态和谜底主题开场。
```

3. 用户提问

输入：

```text
它是车上的东西吗？
```

讲法：

```text
用户发言、车辆状态、环境状态、乘客状态和游戏状态会一起进入 Workflow。
```

4. 触发或等待模拟乘客发言

讲法：

```text
为了让分享体验更真实，其他乘客不是固定剧本，而是由 Workflow 根据当前游戏状态生成一个 passenger_action。
```

5. 触发时间轴事件

讲法：

```text
多模态感知信息通过时间轴自动发生，比如车速变化、天气变化、有人睡着或主驾疲惫。
```

6. 点击或等待 `急刹打断`

讲法：

```text
急刹是高优先级车辆事件，会立即中断游戏。AI 从主持人切换成安全提醒，模拟乘客也停止发言。
```

7. 等待自动恢复

```text
风险解除后，游戏继续，AI 仍然保留上下文。
```

8. 输入答案

输入：

```text
答案是安全带
```

讲法：

```text
猜中后，AI 揭晓谜底并给出情绪价值。后续同一套座舱状态、时间轴和模拟乘客机制可以复用到其他游戏插件。
```

## Workflow 测试

配置真实 Coze/Dify 接口后执行。

### 1. 开场请求

步骤：

1. 点击 `开始模拟`

预期：

- Workflow 收到 `trigger_type = event`
- `event.type = start_game`
- 页面展示 Workflow 返回的 `ai_reply_text`

### 2. 用户提问请求

步骤：

1. 输入：`它是活的吗？`
2. 点击 `发送提问`

预期：

- Workflow 收到 `trigger_type = chat`
- `player_input` 正确
- `game.question_count` 正确
- 页面展示 Workflow 返回结果

### 3. 模拟乘客请求

步骤：

1. 触发模拟乘客发言

预期：

- Workflow 收到 `trigger_type = simulation`
- `interaction.simulation_request.enabled = true`
- Workflow 最多返回一个 `passenger_action`
- 页面正确展示该乘客发言

### 4. 时间轴事件请求

步骤：

1. 启动时间轴
2. 等待一次事件触发

预期：

- Workflow 收到 `trigger_type = event`
- `event.source = timeline`
- `timeline.current_event` 正确
- 页面按返回结果更新

### 5. 急刹请求

步骤：

1. 点击或等待 `急刹打断`

预期：

- 页面立即暂停
- Workflow 收到 `event.type = hard_brake`
- Workflow 返回 `game_status = paused`
- `passenger_action = null`
- 页面保持暂停态
- 前端随后自动恢复游戏状态

### 6. 猜对请求

步骤：

1. 输入当前谜底
2. 点击 `发送提问`

预期：

- Workflow 返回 `is_correct = true`
- Workflow 返回 `show_answer = true`
- 页面揭晓谜底

## 状态组合测试

### 高速安全模式

步骤：

1. 点击车速 `100` 或 `120`

预期：

- 座舱提示高速安全相关状态
- 主驾参与降低
- 模拟乘客不应频繁选择主驾

### 雨天环境

步骤：

1. 点击 `城区雨天白天` 或等待时间轴切换到雨天

预期：

- 右侧背景更新为雨天
- AI 话术提到环境变化
- 待开局时下一题可切到雨天相关谜底

### 乘客睡着

步骤：

1. 让时间轴触发某乘客睡着，或点击 `有人睡着`

预期：

- 对应乘客状态变为睡着
- AI 话术避免 cue 该乘客
- 模拟乘客系统不选择该乘客主动发言

### 快到目的地

步骤：

1. 点击或等待 `快到目的地`

预期：

- AI 进入绝杀局话术
- 游戏保持可继续

## 常见故障与处理

### Workflow 没反应

检查：

- 接口地址是否正确
- 接口是否支持跨域
- 是否需要鉴权
- Workflow 是否返回 JSON

处理：

- 对外体验显示 AI 服务繁忙
- 开发环境可切换 Mock 或查看调试信息

### 模拟乘客太假

检查：

- 是否重复问问题
- 是否连续同一人发言
- 是否太早猜中
- 是否在暂停时发言

处理：

- 调整发言意图权重
- 增加已问问题约束
- 增加座位冷却时间

### 时间轴事件冲突

检查：

- 手动事件是否覆盖时间轴
- 时间轴是否在暂停状态仍触发普通发言
- 高优先级事件是否正确打断

处理：

- 优先保证 P0/P1 事件
- 普通模拟乘客事件在暂停时跳过

### 控制区遮挡

检查：

- 浏览器窗口是否过矮
- 是否低于 1280 x 720

处理：

- 放大窗口
- 使用 16:9 屏幕
